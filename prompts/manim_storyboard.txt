## Your task

This is a **micro-animation** — one atomic mathematical operation, shown with concrete numbers. The rough shot list defines the beat count (2–4 beats). Do NOT add beats beyond what the shot list specifies, do NOT show surrounding context or pipeline.

Expand EACH beat in the rough shot list into one implementation-ready storyboard beat. The rough_shot_list is your contract — implement it faithfully with concrete numbers.

For each beat write:

**Beat N — [one-line title matching the shot_list beat]**
- *Objects*: Exact objects to create with specific numeric values labeled. Prefer node_row for value sequences, heatmap for matrices, bar_chart for distributions.
- *Animation*: Exact animation calls. Show numbers changing — use `always_redraw` + `ValueTracker` for smoothly animating scalar values.
- *Computation spec* (**REQUIRED**): State exactly (a) what values appear on screen as numbers, (b) what transformation animation shows the computation. Choose one: `ValueTracker` (animating a changing scalar), `ReplacementTransform(old, new)` (symbol/formula morphing), `animate_bars` (distribution changing), `interpolate_color` in a loop (weight-proportional coloring), or `Transform(src, dst)` (shape morphing). If none of these apply, rethink the beat — a beat with no transformation is a slideshow frame.
- *Duration*: `self.wait(2.0)` — minimum 2s per beat.
- *Insight*: One sentence — what the viewer understands after this beat.

**Key rule**: Show the computation happening. Values must change on screen. A beat that only calls FadeIn/Write/Create without any transformation will be rejected by the critic.

## Layout helpers — USE THESE FIRST

These are pre-injected into every scene. Use them for all rows, flows, grids, and connections:

```
node_row(labels, colors=None, center=CONTENT_CENTER, spacing=1.5, radius=0.32)
  → (circles: VGroup, texts: VGroup)
  Use for: token rows, node sequences, entity groups (horizontal circles)

side_by_side(stage_labels, stage_colors=None, center=CONTENT_CENTER, box_width=2.6, box_height=1.4, h_spacing=3.8)
  → (boxes: list[VGroup], arrows: list[Arrow])
  Use for: LEFT-TO-RIGHT layouts — encoder/decoder, input/output, any horizontal boxes
  ⚠️ This is the horizontal version. Use when things sit side-by-side.

flow_column(stage_labels, stage_colors=None, center=CONTENT_CENTER, box_width=5.2, box_height=0.68, v_spacing=1.15)
  → (boxes: list[VGroup], arrows: list[Arrow])
  Use for: TOP-TO-BOTTOM pipelines — processing stages stacked vertically
  ⚠️ This is VERTICAL only. Never use it for side-by-side layouts.

connect(obj_a, obj_b, color=WHITE, stroke_width=2, buff=0.1)  → Arrow
  Use for: ALL extra connections between objects. Never use Arrow(LEFT*n, RIGHT*n).

connect_curved(obj_a, obj_b, angle=TAU/4, color=WHITE)  → CurvedArrow

heatmap(matrix, row_labels, col_labels, cell_size=0.62, center=CONTENT_CENTER)  → VGroup
  Use for: attention matrices, weight grids, correlation tables

bar_chart(values, labels, colors=None, max_height=3.2, center=CONTENT_CENTER)
  → (bars, label_texts, baseline)
  + animate_bars(scene, bars, baseline)  — grows bars from baseline
```

## Equations — use MathTex (LaTeX is available)

For ALL mathematical expressions, formulas, and symbols use `MathTex`, not `Text`:
```
MathTex(r"\text{softmax}(x_i) = \frac{e^{x_i}}{\sum_j e^{x_j}}")
MathTex(r"w \leftarrow w - \eta \nabla L")
MathTex(r"\lnot \text{Provable}(n)")
```
- Use `Write(eq)` or `TransformMatchingShapes(eq1, eq2)` to animate
- Use `eq.get_part_by_tex("symbol")` to highlight sub-expressions
- For step-by-step formula evaluation: create multiple `MathTex` objects and `ReplacementTransform` between them

Use `Text()` only for plain English labels — NEVER for math or LaTeX (use `MathTex`). Every `Text()` MUST have explicit `font_size` (default 48 is huge): title=34, body=24, annotations=18.

## Verified animation calls ONLY

Creation: `Create`, `Write`, `FadeIn`, `FadeOut`, `DrawBorderThenFill`
Transform: `Transform`, `ReplacementTransform`, `TransformMatchingShapes`
Motion: `obj.animate.shift()`, `obj.animate.move_to()`, `obj.animate.scale()`, `obj.animate.set_color()`
Highlight: `Indicate`, `Flash`, `Circumscribe`, `SurroundingRectangle`
Composite: `LaggedStart`, `AnimationGroup`, `Succession`
Tracker: `ValueTracker`, `always_redraw`

**DO NOT use**: `GrowFromCenter`, `Pulse`, `ShowCreation`, `ShowPassingFlash`, `apply_complex_function`, `GrowArrow` (use `Create` instead), or any other animation not listed above.

## Valid colors ONLY

`BLUE/BLUE_B/BLUE_C/BLUE_D`, `GREEN/GREEN_B/GREEN_C`, `RED`, `ORANGE`, `YELLOW`, `WHITE`, `GREY/GREY_B/GREY_C/GREY_D`, `TEAL`, `PURPLE`

**DO NOT use**: `LIGHT_RED`, `DARK_BLUE`, `LIGHT_GREEN`, or any `LIGHT_*`/`DARK_*` variant.

## Spatial rules

- Title `Text("{{CONCEPT_NAME}}", font_size=34).to_edge(UP)` persists throughout — never remove it
- All content anchored to `CONTENT_CENTER` (= `DOWN * 0.5`) — no bare coordinate vectors
- Derive positions from objects: `.next_to(other, DOWN, buff=0.3)` ✓ — not `.move_to([1.2, -0.5, 0])` ✗
- Fade out content before introducing new content in the same region

- Colors are used consistently: one color per concept throughout
