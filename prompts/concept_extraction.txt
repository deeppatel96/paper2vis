## The atomic test — apply this strictly

Ask: "Can I show THIS paper's specific operation with concrete values changing on screen in one shot?"

**REJECT these:**
- Concepts not mentioned in the section text
- Vague overviews ("the proof works by...", "the system uses...")

- Concepts borrowed from other papers or your training data
- Concepts where you cannot name at least one specific value, symbol, or formula from the text

**ACCEPT these — atomic operations with concrete values from the actual paper:**

## What to animate: operations with values, not labels in boxes

Good: a formula with numbers substituted in, a value changing step by step, a logical substitution shown symbolically, a geometric transformation applied to coordinates.

Bad: a box labeled "Encoder" → another box labeled "Decoder". A diagram of system components. A timeline of paper history.

## Visual types — pick the best fit for THIS paper's content

1. **equation_transform** — a formula computing step-by-step (most papers have this)
2. **geometric** — a vector, shape, or space being transformed
3. **number_flow** — values propagating through nodes (3–5 nodes max)
4. **weight_update** — a parameter animating from one value to another
5. **matrix_op** — a small matrix (≤4×4) computation
6. **diagram** — last resort: only when the concept is genuinely structural

## Shot list rules — 2 to 4 beats

Each shot_list has **2–4 beats** depending on complexity:
- Simple computations (one formula, one result): **2 beats**
- Multi-step derivations or comparisons: **3 beats**
- Full worked examples with intermediate steps: **4 beats**

Beat guidelines:
- **Beat 1**: Show the initial state with THIS paper's specific values labeled
- **Middle beats** (if needed): Show intermediate transformation steps with concrete values at each step
- **Final beat**: Show the result / conclusion with concrete values; add a text label summarizing what the viewer just saw

**Good shot list for a logic paper (2 beats):**
```
1. "Show formula '¬Provable(n)' in MathTex centered. Label it 'The Gödel sentence G'. Show Gödel number n=2486 in a box below."
2. "Animate: highlight 'Provable(n)' in RED → show it evaluating to FALSE. Then highlight the outer '¬' turning the result to TRUE. Text appears: 'G is true but not provable.' Viewer sees the self-referential paradox."
```

**Good shot list for a multi-step derivation (3 beats):**
```
1. "Show attention formula: score(Q, K) = QKᵀ / √d_k. Label Q=[1,0], K=[0.8,0.6], d_k=2."
2. "Animate: compute dot product QKᵀ = 0.8, then divide by √2 ≈ 1.41, result = 0.57 appears in a highlighted box."
3. "Apply softmax: show [0.57, 0.23, 0.19] → probabilities [0.44, 0.30, 0.26] appearing. Label 'token 1 gets 44% of attention'."
```
