## Layout helpers (auto-injected — use freely, NEVER redefine)

⚠️ CRITICAL: These functions are already in the execution environment. Do NOT define `def node_row(...)`, `def flow_column(...)`, `def connect(...)`, `def connect_curved(...)`, `def heatmap(...)`, `def bar_chart(...)`, or `def animate_bars(...)` anywhere in your code — not inside `construct()`, not at module level. Redefining them shadows the correct implementation and breaks the scene.

**Prefer these over manual positioning whenever possible** — they eliminate the most common source of arrow and centering mistakes.

```
node_row(labels, colors=None, center=CONTENT_CENTER, spacing=1.5, radius=0.32)
  → (circles: VGroup, texts: VGroup)
  Horizontal row of filled circles with centered labels. Positions derived from center.
  Example: circles, texts = node_row(["Q","K","V"], [GREEN, ORANGE, RED], center=CONTENT_CENTER)

side_by_side(stage_labels, stage_colors=None, center=CONTENT_CENTER, box_width=2.6, box_height=1.4, h_spacing=3.8)
  → (boxes: list[VGroup], arrows: list[Arrow])
  Horizontal LEFT-TO-RIGHT row of RoundedRectangles. Use for encoder/decoder, input/output, any side-by-side layout.
  Example: boxes, arrows = side_by_side(["Encoder","Decoder"], [BLUE,GREEN], center=CONTENT_CENTER)
  ⚠️ Use this (not flow_column) whenever boxes should sit left-to-right.

flow_column(stage_labels, stage_colors=None, center=CONTENT_CENTER, box_width=5.2, box_height=0.68, v_spacing=1.15)
  → (boxes: list[VGroup], arrows: list[Arrow])
  VERTICAL top-to-bottom stack of RoundedRectangles. Use only for vertical pipelines.
  Example: boxes, arrows = flow_column(["Embed","Attention","FFN","Output"], [BLUE,TEAL,TEAL,ORANGE], center=CONTENT_CENTER)
  ⚠️ flow_column is VERTICAL. Never use it for side-by-side (left-right) layouts.

connect(obj_a, obj_b, color=WHITE, stroke_width=2, buff=0.1)
  → Arrow
  Arrow between two mobjects using their actual boundary points (.get_right/.get_left etc).
  Automatically picks horizontal or vertical axis. ALWAYS use this instead of Arrow(LEFT*n, RIGHT*n).
  Example: arr = connect(encoder_box, decoder_box, color=BLUE)

connect_curved(obj_a, obj_b, angle=TAU/4, color=WHITE, stroke_width=2)
  → CurvedArrow
  Curved arrow between two mobjects using their actual boundary points.
  Example: fb = connect_curved(decoder_box, encoder_box, angle=TAU/3, color=RED)

heatmap(matrix, row_labels=None, col_labels=None, cell_size=0.62, center=CONTENT_CENTER)
  → VGroup
  Attention-style colored grid. matrix is 2D list of floats in [0,1].
  Example: grid = heatmap([[0.1,0.7],[0.4,0.4]], row_labels=["Q1","Q2"], col_labels=["K1","K2"], center=CONTENT_CENTER)

bar_chart(values, labels, colors=None, max_height=3.2, bar_width=0.85, spacing=1.35, center=CONTENT_CENTER)
  → (bars: list, label_texts: list, baseline: Line)
  Bar chart with all positions derived from center. Use animate_bars() to grow them.
  Example: bars, lbls, baseline = bar_chart([28.4,41.8], ["RNN","Transformer"], [BLUE,ORANGE], center=CONTENT_CENTER)

animate_bars(scene, bars, baseline, lag_ratio=0.15, run_time=0.5)
  Grows each bar from the baseline upward. Call after Create(baseline).
  Example: animate_bars(self, bars, baseline)
```

## Reference example — quality target

This is a complete, working scene. Match this level of specificity, layout discipline, and animation depth:

```python
from manim import *
import numpy as np

class SelfAttentionScene(Scene):
    def construct(self):
        title = Text("Self-Attention  ·  Attention Is All You Need", font_size=32).to_edge(UP)
        self.play(FadeIn(title))
        self.wait(0.5)

        # Beat 1: token row using node_row helper
        tokens = ["The", "cat", "sat", "on", "mat"]
        circles, labels = node_row(
            tokens, [BLUE_C] * 5,
            center=CONTENT_CENTER + UP * 1.85, spacing=1.65, radius=0.32,
        )
        row_lbl = Text("Input sequence", font_size=15, color=GREY_B).next_to(circles, UP, buff=0.18)
        self.play(LaggedStart(*[Create(c) for c in circles], lag_ratio=0.12))
        self.play(LaggedStart(*[FadeIn(l) for l in labels], lag_ratio=0.12))
        self.play(FadeIn(row_lbl))
        self.wait(0.8)

        # Beat 2: weighted attention arrows from query token
        weights = np.array([0.03, 0.14, 0.68, 0.09, 0.06])
        q_ring = Circle(radius=0.44, color=YELLOW, stroke_width=3).move_to(circles[2])
        q_lbl = Text("Query: 'sat'", font_size=18, color=YELLOW)
        q_lbl.move_to(CONTENT_CENTER + UP * 0.60)
        self.play(Create(q_ring), FadeIn(q_lbl))

        sat_c = circles[2].get_center()
        w_objs = VGroup()
        w_txts = VGroup()
        for i, w in enumerate(weights):
            col = interpolate_color(GREY_C, YELLOW, min(1.0, w * 2.6))
            sw = 1.2 + w * 15
            if i == 2:
                arc = Arc(radius=0.32, start_angle=PI*0.1, angle=PI*1.8,
                          arc_center=sat_c + UP*0.48, stroke_width=sw, color=YELLOW)
                w_objs.add(arc)
            else:
                ln = Line(sat_c, circles[i].get_center(), stroke_width=sw, color=col)
                ln.set_stroke(opacity=0.85)
                w_objs.add(ln)
            wt = Text(f"{w:.2f}", font_size=14, color=col).next_to(circles[i], DOWN, buff=0.14)
            w_txts.add(wt)
        self.play(LaggedStart(*[Create(o) for o in w_objs], lag_ratio=0.1))
        self.play(LaggedStart(*[FadeIn(t) for t in w_txts], lag_ratio=0.1))
        self.wait(2.0)
        self.play(FadeOut(w_objs), FadeOut(w_txts), FadeOut(q_ring), FadeOut(q_lbl))

        # Beat 3: full attention heatmap using heatmap helper
        attn = [
            [0.55, 0.20, 0.12, 0.07, 0.06],
            [0.18, 0.48, 0.22, 0.07, 0.05],
            [0.03, 0.14, 0.68, 0.09, 0.06],
            [0.05, 0.10, 0.28, 0.45, 0.12],
            [0.04, 0.07, 0.18, 0.24, 0.47],
        ]
        heat = heatmap(attn, row_labels=tokens, col_labels=tokens,
                       cell_size=0.62, center=CONTENT_CENTER + DOWN * 0.55)
        heat_ttl = Text("Full 5×5 attention matrix", font_size=18)
        heat_ttl.move_to(CONTENT_CENTER + UP * 1.1)
        self.play(FadeIn(heat_ttl))
        self.play(FadeIn(heat), run_time=0.8)
        self.wait(1.5)

        # Beat 4: highlight sat row + final message
        hmap_center = CONTENT_CENTER + DOWN * 0.55
        sat_row_y = hmap_center[1] + ((5-1)/2 - 2) * 0.62
        row_rect = Rectangle(width=5*0.62+0.08, height=0.62-0.06,
                              color=YELLOW, stroke_width=2.5, fill_opacity=0)
        row_rect.move_to(RIGHT * hmap_center[0] + UP * sat_row_y)
        sat_note = Text("'sat' attends most to itself (0.68)", font_size=17, color=YELLOW)
        sat_note.next_to(heat, DOWN, buff=0.28)
        self.play(Create(row_rect))
        self.play(FadeIn(sat_note))
        self.wait(2.5)
```

## Implementation rules

**Follow the storyboard beat-by-beat.** Do not invent new content or skip beats.

**MANDATORY: Every scene must contain at least one dynamic transformation.** An animation where every object only `FadeIn`s or `Write`s is a slideshow, not an animation — it will be rejected. You MUST use one or more of:
- `Transform(src, dst)` or `ReplacementTransform(src, dst)` — morph one object into another
- `TransformMatchingShapes(eq1, eq2)` — morph formulas step-by-step
- `ValueTracker` + `always_redraw(lambda: ...)` — animate a continuously changing value
- `obj.animate.set_color(...)` / `.animate.scale(...)` / `.animate.shift(...)` — motion on existing objects
- `interpolate_color(c1, c2, t)` inside a loop — visual transition proportional to data

**FORBIDDEN ANTI-PATTERNS — these will score ≤ 5 and fail the critic:**
- `self.play(FadeIn(result_box))` to show a computation result → use `ReplacementTransform(input_obj, result_obj)` instead
- `self.play(FadeIn(bars))` for a distribution → use `animate_bars(self, bars, baseline)` so bars grow from zero
- `self.play(Write(formula))` then nothing → follow with `TransformMatchingShapes(formula, next_formula)` showing evaluation steps
- Displaying a matrix/heatmap with `FadeIn` only → add `Indicate(cells[i][j], ...)` to highlight the key value
- A beat where ALL calls are FadeIn/Write/Create — at least one call must be Transform/ReplacementTransform/ValueTracker/animate_bars/interpolate_color
- Using `Text(r"\frac{1}{2}")` for math → use `MathTex(r"\frac{1}{2}")` always

**Static reveals are insufficient for the computation beat.** If Beat 2 shows a result (e.g. a transformed value, a computed output, an encoded sequence), animate HOW it is computed:
- Numbers changing: use `ValueTracker` + `always_redraw`, or `ReplacementTransform(old_num, new_num)`
- Symbol→number encoding: use `LaggedStart([ReplacementTransform(sym.copy(), num) for sym, num in zip(syms, nums)], lag_ratio=0.3)`
- Formula evaluation: use multiple `MathTex` objects with `TransformMatchingShapes` between steps
- Bar heights: use `animate_bars(self, bars, baseline)` — never `FadeIn(bars)`

**Spatial positioning — strict rules:**
- NEVER pass bare direction vectors to Arrow/Line: `Arrow(LEFT*2, RIGHT*3)` ✗
- ALWAYS use boundary methods or helpers: `connect(obj_a, obj_b)` ✓  or `Arrow(obj_a.get_right(), obj_b.get_left())` ✓
- NEVER use `move_to([1.2, -0.5, 0])` with numeric lists ✗
- Derive all positions from other objects: `obj.next_to(other, DOWN, buff=0.3)` ✓  or `center + RIGHT*x + UP*y` ✓

**Title zone — strictly enforced:**
- Always add title: `title = Text("...", font_size=34).to_edge(UP)` — show it at the start and never remove it.
- The title occupies the top ~1.2 units of the canvas. ALL other objects must stay below `CONTENT_TOP` (y ≤ 2.7).
- Use `CONTENT_CENTER` as the anchor for your main content (it equals `DOWN * 0.5`, the vertical midpoint of the safe zone).
- Both `CONTENT_TOP` and `CONTENT_CENTER` are auto-injected constants — use them directly, no import needed.

**Code requirements:**
- Imports: `from manim import *`, `import numpy as np` only. Do NOT import layout_helpers — it's auto-injected.
- One class inheriting `Scene`, named descriptively (e.g. `SelfAttentionScene`).
- Complete and runnable — no `...`, no undefined variables.

**Verified animation calls only** (others may not exist):
- Creation: `Create`, `Write`, `FadeIn`, `FadeOut`
- `DrawBorderThenFill` ONLY on a single VMobject (Circle, Rectangle, Polygon) — NEVER on VGroup, list, or layout helper output. Use `FadeIn` or `Create` for groups.
- Transform: `Transform`, `ReplacementTransform`, `TransformMatchingShapes`
- Motion: `obj.animate.shift()`, `obj.animate.move_to()`, `obj.animate.scale()`, `obj.animate.set_color()`
- Highlight: `Indicate`, `Flash`, `Circumscribe`, `SurroundingRectangle`
- Composite: `LaggedStart`, `AnimationGroup`, `Succession`
- Tracker: `ValueTracker`, `always_redraw`
- Do NOT use: `ShowCreation`, `Pulse`, `GrowFromCenter`, `ShowPassingFlash`, `GrowArrow`
- Colors: `BLUE/BLUE_B/BLUE_C/BLUE_D`, `GREEN/GREEN_B/GREEN_C`, `RED`, `ORANGE`, `YELLOW`, `WHITE`, `GREY/GREY_B/GREY_C/GREY_D`, `TEAL`, `PURPLE` — no `LIGHT_*` or `DARK_*`

**Undefined constants that do NOT exist in Manim (crash immediately):**
- `LEFT_CENTER`, `RIGHT_CENTER`, `TOP_CENTER`, `BOTTOM_CENTER` — use `CONTENT_CENTER + LEFT*n` etc.
- Use `Scene` (not `MovingCameraScene`) unless you explicitly need camera movement. Never call `self.camera.frame` in a regular Scene.

**Division safety:** When rescaling bars or lengths by a value that could be zero, guard with `max(val, 0.001)` before dividing.

**Equations — use MathTex (LaTeX is installed and available):**
- `MathTex(r"\frac{e^{x_i}}{\sum_j e^{x_j}}")` for rendered math — PREFERRED over Unicode Text
- `TransformMatchingShapes(eq1, eq2)` to morph one formula into another
- `eq.get_part_by_tex("x_i")` to isolate and highlight sub-expressions
- For animated values: `always_redraw(lambda: MathTex(f"w = {t.get_value():.2f}"))`
- `get_x_axis_label`, `get_y_axis_label` → `Text(...).next_to(axes.get_x_axis(), RIGHT, buff=0.15)`

**CRITICAL — Text() vs MathTex():**
- `Text()` is for plain English ONLY: `Text("Encoder", font_size=24)` ✓
- ANY string with subscripts, superscripts, fractions, or variable names MUST use `MathTex(r"...")`
  - `Text("d_A")` ✗ → `MathTex(r"d_A")` ✓  (underscore renders as literal underscore in Text!)
  - `Text("x^2")` ✗ → `MathTex(r"x^2")` ✓
  - `Text("O(n^2)")` ✗ → `MathTex(r"O(n^2)")` ✓
  - `Text("r·d_B")` ✗ → `MathTex(r"r \cdot d_B")` ✓
  - `Text("α, β, γ")` ✗ → `MathTex(r"\alpha, \beta, \gamma")` ✓
- Every `Text()` call MUST include `font_size` explicitly (default is 48 which is huge): title=34, body=24, annotations=18, tiny=14

**Animation quality:**
- `LaggedStart([FadeIn(x) for x in items], lag_ratio=0.2)` for staggered reveals
- `ValueTracker` + `always_redraw` for continuously changing quantities
- `rate_func=smooth` or `rush_from` for natural motion
- `self.wait(1.5)` or longer between beats
